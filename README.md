# Исправление утечек памяти в clib

Найдены и исправлены две утечки памяти в функции `clib_package_new` (файл `src/common/clib-package.c`).

## Найденные утечки

1. **Утечка при обработке флагов**  
   В цикле по массиву `flags` при ошибке `asprintf` память, выделенная под строку `flag`, не освобождалась. Исправлено добавлением `free(flag)` перед переходом к `cleanup`.

2. **Утечка при обработке списка исходных файлов (src)**  
   В цикле по массиву `src` при ошибках (NULL-указатель или неудачное добавление в список) память под `file` не освобождалась. Исправлено добавлением `free(file)` в соответствующих ветках.

## Исправления

Патч: `fix-leak.patch`

## Проверка

Анализ проведён статически (в среде MSYS2, где valgrind не поддерживается). Обе утечки устранены. После применения патча утечки исчезают.
